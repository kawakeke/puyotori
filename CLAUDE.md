# puyotori プロジェクト

しりとりをテーマにしたブラウザゲーム集。依存ライブラリなし・ビルド不要の単一HTMLファイル構成。

## 運用ルール(ユーザーとの取り決め)

- **ユーザーの指示・好み・方針が出たら、都度このCLAUDE.mdに追記すること。**

## ゲームの方向性(2026-07 ユーザー決定)

- **ターゲットは大人。** 子ども向けトーンは廃止。
- 目指す味: **ビター/シュール**。参照作品はメイドインワリオ、せがれいじり。「にやり」とさせる。
- UI文言は真顔のデッドパン(会社員的な言い回し)。例: 難易度=接待/定時/激務、獲得=「回収」。
- CPUの呼称は「**相手**」。(「先方」は違和感があるとのユーザー指摘で変更済み 2026-07-14)
- 画面フロー: ゲーム中は上部に「降参」ボタン(押すと1敗カウントでTOPへ)。結果画面は「もういちど(同設定で即再戦)」と「TOPへ戻る」の2ボタン。通算成績「◯勝◯敗◯分」をlocalStorage(`shiritori3d_record`)に保存し、TOP画面と結果画面に表示。
- **「ん」トラップ=人生の落とし穴**というメタファー(誤送信・終電・左遷・割り勘など)。しりとりのルールと大人の悲哀を重ねるのがコアの面白さ。
- しりとりベースで「なるほど」と唸らせる仕掛けを優先する。
- **ストーリー(2026-07-16 ユーザー決定): 宇宙人が日本に潜入するための「研修」**。プレイヤー=潜入要員の研修生、ライバルUFO=**同期**(「相手」から変更)、「ん」=**正体露見**、残り10秒×2=夜明けが近い。初回起動時に【辞令】イントロ(UFOが🗾に降下するCSSアニメ+真顔の辞令文、localStorage `shiritori3d_introSeen`、TOPの「▶辞令を読み直す」で再表示)。結果画面=修了/再研修/判定保留。眼下に夜の地球グロー(青+街明かり)。

## ファイル

- `index.html` — **しりとりハント3D**(メイン開発中)。3D空間に浮かぶアイテムを捕まえる早い者勝ちしりとり。CPU(UFO)との対戦。※GitHub PagesのルートURLで直接遊べるよう `shiritori3d.html` から改名(2026-07-15)。
- `puyoritori.html` — ぷよりとり(ぷよぷよ×しりとり)。初期の別ゲーム。基本触らない(旧 `index.html`)。
- 動作確認: `.claude/launch.json` の static サーバー(python3 http.server, port 8731)で `/` を開く。

## しりとりハント3D の設計

### 仕組み
- **2層レンダリング(2026-07-14 B案採用)**: 世界(星・星雲・床グリッド・アイテム・UFO)は**自作の生WebGL**、UI(網カーソル・なわ・メッセージ・ビーム・ヒント)はCanvas 2Dオーバーレイ。three.js等は不使用(外部依存なし方針を維持)。
- アイテムは方位角az/仰角el/距離rの球面座標で管理。2D側`project()`とGL側`vpMatrix()`は厳密に同じ投影(FOV/focal共有)なので当たり判定・装飾の位置が一致する。ドラッグで視点回転。
- GL側: アイテムは256px canvasに絵文字/漢字+ラベル+バッジを焼いたテクスチャのビルボード(`paintItemTexture`、ボーナス時は自動再生成)。距離フォグ、UFOはランバート照明付き回転体メッシュ、トレイル/星雲は加算グロー、床はワイヤーフレームグリッド。画面シェイクはカメラの揺れ(`shakeYaw/shakePitch`)として両層に適用。
- 注意: 長い単語のラベルは`measureText`でテクスチャ幅238pxに収める。タブ非表示でロードされるとviewportが0になるためループ内でサイズ追従resize()あり。
- 辞書は3パック(`PACKS`)。表示順・表示名は左から「定番/漢字/**サラリーマン**」(2026-07-17 レビューで「ビター」から改名): `DICT`(定番えもじ 約215語) / `KANJI_DICT`(難読漢字 約215語、読みでしりとり) / `BITTER_DICT`(サラリーマン 約230語、デフォルト)。サラリーマンには**おじさんビジネス用語**を多数含める方針(えいや・がっちゃんこ・アジェンダ・たたき台・ポンチ絵・テレコ・横串・よしなに・一丁目一番地、トラップ: ざっくばらん・がらがらぽん・けつかっちん・しゃんしゃん)。
- ビター系の演出: アイテムは網カーソルが近づくと逃げて震える(長い言葉ほどすばしっこい)。トラップは時々💦をかく(さりげないテル)。UFOはトラクタービームで吸引、獲得アイテムはUFOに吸い込まれる。トラップで画面シェイク。45秒/30秒で真顔の小言(小フォントのquip)、トラップ後もTRAP_QUIPSからひとこと。結果画面にbitterComment(スコア差で皮肉な講評)。
- 「ん」で終わる単語は自動的にトラップ(`trap: w.endsWith("ん")`)。捕ると-15点+2秒スタン。
- 得点 = 文字数×5。長い単語ほど小さく(`sizeF`)・速くドリフト(`spd`)するリスクリワード設計。
- 連続ゲットでコンボ(+5ずつ、最大+25)。残り10秒はボーナスタイム(全得点×2)。
- お題に対し「**ふつう正解2・高得点正解(5文字以上)2・「ん」トラップ2**」を常時目標に補充(`ensureAnswers`、辞書に足りない文字は在庫ぶんだけ)(2026-07-17 レビュー反映)。辞書が尽きたらお題を自動シャッフル(`reshuffleKana`)。
- **開始のお題はランダム**(毎回「り」固定を廃止 2026-07-17)。続く言葉が3語以上ある「開始語」を辞書から選び、その最終文字から開始。
- アイテムの湧き位置は**完全ランダム**(2026-07-16 ユーザー指定。当初の「重ならない場所を選ぶ」方式は廃止)。
- カーソルは捕虫網表現(`drawNet`)。捕獲方法は2つ: ①じっと長押し(650ms)、②**なわ捕獲**=**ダブルクリック(タップ)して押したまま**円で囲う(`lassoMove`: ストロークの巻きつき角が5.3rad超で捕獲、一筆で連続捕獲可)。ふつうのドラッグは常に視点回転。※当初の「アイテム付近から押したら囲いモード」は視点回転と紛らわしいとのユーザー指摘で、ダブルクリック起動に変更(2026-07-14)。
- なわで複数囲んだ場合は**囲みにお題の言葉が入っていればOK**(ハズレは素通し・ペナルティなし。お題一致のトラップだけは発動)(2026-07-17 レビュー反映)。
- 結果画面に「次のステップ」導線: 接待クリア→定時、定時クリア→激務、激務クリア→次パックへ転属。敗北時は降格の打診(激務→定時など)。真顔のコメント付き(2026-07-17)。
- UI基調: **未来的ホログラム調**(シアン発光・角カットclip-path・走査線)。最先端の研修シミュレータで「がっちゃんこ」を学ぶシュールな対比が狙い(2026-07-17)。
- ヒント: 4秒無操作で答えのある方向の画面隅がほんのり金色に明滅(控えめに、が方針)。
- CPUは難易度で反応速度・移動速度・狙い(`pick`: min=安い語/rand/max=高得点語)が変わる。

## 方針メモ

- 単一HTML・外部依存なしを維持する(オフラインでも動くこと)。
- 演出は「控えめ」寄りをユーザーが好む(派手な矢印ヒント→隅の淡い光に変更した経緯あり)。
- **説明文はSP(スマホ)でも成立する短さにする。** スタート画面のルールは4行に短縮した経緯あり(2026-07-16)。長文の説明はNG。
- UI文言はひらがな中心(子どもも遊べるトーン)。漢字モードのみ大人向け。
- 辞書に単語を足すときは「ん」で終わる語がトラップになること、続く文字の単語が辞書に存在するか(死に文字はreshuffleで救済されるが最小限に)を意識する。
